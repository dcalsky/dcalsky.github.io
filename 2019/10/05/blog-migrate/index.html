博客迁移至zeit/now

Oct 5, 2019 | 技术

Github Page访问比较慢, 在有些地区还会被墙. 必须要想一个出路了.

因为之前(2018年)写过一期文章评估了一下各大服务商提供的Static page service的性能与可用性. 虽然最终选择了Github Page, 但它还是不尽如人意的.

在寻找新的Service时我定了几点小目标:

  1. 部署简单, 能够配合相关CI/CD
  2. 免费(或者价格低廉): 因为博客一直要开着的, 长期费用吃不消
  3. 访问流畅,速度要快: 起码要比Github Page快
  4. Custom Domain

所以新的服务不能比Github Page操作难, 也不能比它慢.

zeit/now

很久以前就听过now的大名了, 但当时还是热衷自己部署应用到VPS上, 并且深深怀疑AWS Lambda能否在国内访问, 所以就一直没试用过.

现在要换服务了, 第一时间就想起了免费的他.🤣 初步看了一下满足了1,2的要求, 当时3, 4还需要自己亲自测试. 于是我跟着教程部署了一个简单的网站, 用手机访问发现意外的流畅, 比打开Github Page要快一些, 但是还是需要进一步控制变量的.

配合Hexo

因为我的博客是基于Hexo的, 而且我也很习惯了之前部署的操作, 所以我希望在尽可能不改变这个操作流程的基础上去适配zeit/now. 很多人跟我一样都是用hexo-deployer-git通过hexo d -g命令自动把public/下的资源部署到username.github.io仓库下的.

但是, 要出发now的CI就必须要在仓库的根目录有一个now.json配置文件, 所以我需要在hexo应用下的source目录中加上now.json文件, 这样在hexo generate阶段就会把这个文件复制到public/目录下了. 这样一来, now.json就会出现username.github.io仓库的根目录下, 每次更新博客都会触发now的deploy event, 网站就会自动更新.
