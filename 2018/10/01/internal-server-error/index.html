页面无法访问 I tell you

Oct 1, 2018 | 技术

Web开发，无论是本地调试开发还是最终部署上线，都可能遇到页面无法访问的情况。最快速的方法肯定是请个老法师帮忙把把关，其次都是靠自己总结的经验了…

404 Checklist

  • Static files没放在目录里，或者路径出错。
  • Web server未能成功代理Staticfiles，可能是代码写错了，或者是它压根没找到。
  • Url填错了

404之外的Checklist

  • 端口号与hostname
  • 端口安全组设置
  • 针对特定端口防火墙放行
  • 查看Web server的log，检查报错信息
  • 远程主机在国外，是不是网络连接问题
  • DNS设置正确且解析完毕
  • Ping hostname能成功吗？不成功就是网络连接问题
  • 查看Web application接受的request以及报错信息
  • 生产环境下，Web server是否监听的是0.0.0.0下的端口
  • Https证书获取与解析是否遇到了麻烦
  • Web server(Nginx、Apache)配置正确了吗
